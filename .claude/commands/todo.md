# TODO 管理技能

管理项目根目录的 `TODO.md` 文件。用于快速记录临时想到的想法、待办事项、改进点。

## 核心原则

- **快速记录，不深入** — 简要澄清 TODO 的意图即可，不要展开设计或开始开发
- **不触发 superpowers 流程** — 这是记录阶段，不是实施阶段
- **保持简洁** — 每条 TODO 控制在 1-3 行描述

## 操作

根据用户输入判断操作类型：

### 添加 TODO（默认操作）

触发：`/todo 一些想法` 或 `/todo add 一些想法`

1. 读取 `TODO.md` 当前内容
2. 如果用户描述模糊，**最多问 1 个澄清问题**（不要连续追问）
3. 判断优先级（默认 🟢，用户可指定）
4. 追加到 `TODO.md` 末尾，格式：

```markdown
## 🟢 标题
- 简要描述
- 相关模块：xxx（如果能判断）
- 2026-03-29
```

5. 确认已添加，显示 TODO 标题

### 列出 TODO

触发：`/todo list` 或 `/todo ls`

1. 读取 `TODO.md`
2. 以简洁列表展示所有 TODO（优先级 + 标题 + 日期）

### 完成 TODO

触发：`/todo done <关键词>` 或 `/todo done <序号>`

1. 读取 `TODO.md`
2. 匹配对应 TODO（按标题关键词或出现顺序）
3. 在标题后追加 ✅，移动到文件底部的「已完成」区域
4. 如果「已完成」区域不存在，创建它：

```markdown
---

## 已完成

### ✅ 标题
- ...
- 完成于 2026-03-29
```

### 删除 TODO

触发：`/todo rm <关键词>` 或 `/todo remove <序号>`

1. 读取 `TODO.md`
2. 匹配并删除对应 TODO 条目
3. 确认已删除

## 优先级速记

用户可以在描述中用以下方式指定优先级：
- `!` 或 `紧急` → 🔴
- `!!` 或 `重要` → 🟡
- 默认 → 🟢
